# 模组下载流程

本文档描述 MOneClickDownloads 中模组下载的完整流程，包括版本筛选、冲突检测、依赖递归下载、事务回滚等机制。

## 各组件职责

### LocalModInventory（本地模组清单工具）

**文件**: `MOneClickDownloads.Service/LocalModInventory.cs`

**职责**: 扫描指定文件夹内的所有 `.jar` 文件，利用 `IModAnalysisService` 提取模组元数据，构建本地已安装模组的索引。

**构造函数**:
```csharp
public LocalModInventory(string folderPath, IModAnalysisService analysisService)
```

**核心方法**:

| 方法 | 说明 |
|------|------|
| `ScanAsync()` | 扫描文件夹内所有 `.jar` 文件，构建索引 |
| `FindByModId(modId)` | 按模组 ID 查找（忽略大小写） |
| `GetModFilePath(modId)` | 获取已安装模组的文件路径 |
| `ContainsMod(modId)` | 检查是否已安装指定模组 |

**数据流**:
```
文件夹路径 → 枚举 *.jar → 逐个调用 AnalyzeAsync → 按 ModId 索引到字典
```

### ModDownloadService（模组下载服务）

**文件**: `MOneClickDownloads.Service/ModDownloadService.cs`

**职责**:
- 管理整个下载生命周期
- 冲突检测与回调
- 版本号比较（SemVer）
- 依赖递归下载
- 事务性回滚

### ModConflictDialog（冲突对话框）

**文件**: `MOneClickDownloads.App/Views/ModConflictDialog.axaml` + `.cs`

**职责**: 在 UI 层显示冲突详情，让用户选择处理方式。

## 冲突检测详细流程

### 1. 创建 LocalModInventory

在 `ExecuteDownloadTransactionAsync` 中，若 `conflictCallback` 不为 null，则创建 `LocalModInventory` 并扫描：

```csharp
LocalModInventory? inventory = null;
if (conflictCallback != null)
{
    inventory = new LocalModInventory(saveDirectory, _analysisService);
    await inventory.ScanAsync();
}
```

### 2. 检测冲突（DetectConflict）

对每个待下载文件，调用 `DetectConflict` 方法：

```
ModrinthVersion.ProjectId → LocalModInventory.FindByModId()
  ├─ 未找到 → 返回 null（无冲突）
  └─ 找到 → 比较版本号
       ├─ 版本相同 → SameModExists
       ├─ 本地 > 新 → HigherVersionExists
       └─ 本地 < 新 → LowerVersionExists
```

### 3. 回调用户决定

```csharp
var conflict = DetectConflict(inventory, ver, file);
if (conflict != null)
{
    var resolution = await conflictCallback(conflict);
    // 处理 resolution...
}
```

### 4. 执行冲突解决策略

| 策略 | 行为 |
|------|------|
| **Skip** | 跳过该文件，不下载 |
| **Replace** | 删除本地旧文件，下载新文件到同一路径 |
| **KeepBoth** | 新文件添加版本后缀（如 `mod-2.0.0.jar`），保留旧文件 |

## 冲突解决策略说明

### Skip（跳过）
- 保留本地已有的模组文件
- 新版本不会被下载
- 适用于：用户不想升级时

### Replace（替换）
- 删除本地已有的模组文件
- 下载新版本到相同路径
- 适用于：用户确认要升级/降级时

### KeepBoth（保留两者）
- 本地文件保持不变
- 新文件以 `文件名-版本号.jar` 格式保存
- 适用于：用户想同时保留两个版本（如测试兼容性）

## 版本比较规则（SemVer）

比较逻辑在 `ModDownloadService.CompareVersions` 中实现：

1. **去除 build metadata**：移除 `+` 及其后的部分（如 `1.0.0+build.123` → `1.0.0`）
2. **解析为数字序列**：`1.20.1` → `[1, 20, 1]`
3. **去除 pre-release 标识**：`1.0.0-beta` → 取 `1.0.0` 部分
4. **逐级比较**：主版本 → 次版本 → 修订版本
5. **回退**：无法解析为数字时，使用字符串 OrdinalIgnoreCase 比较

**比较示例**:
| 版本A | 版本B | 结果 |
|-------|-------|------|
| `1.20.1` | `1.20.0` | A > B |
| `2.0.0` | `1.99.99` | A > B |
| `1.0.0+build.1` | `1.0.0+build.2` | A == B（忽略 build metadata） |
| `1.0.0-beta` | `1.0.0` | A < B（去掉 pre-release 后数字相同，回退字符串比较） |

## 事务与回滚机制

### 事务范围

每次调用 `ExecuteDownloadTransactionAsync` 构成一个事务：
- **维护文件列表**：`downloadedFiles` 记录所有已下载文件路径
- **维护项目集合**：`downloadedProjects` 防止循环依赖

### 回滚触发条件

| 触发条件 | 行为 |
|----------|------|
| `OperationCanceledException` | 用户取消 → 回滚 |
| 其他异常 | 下载失败 → 回滚 |
| `DownloadException` | 已由内部处理，不二次回滚 |

### 回滚过程

```csharp
private static void RollbackDownloadedFiles(List<string> filePaths)
{
    foreach (var filePath in filePaths)
    {
        TryDeleteFile(filePath); // 尽力删除，忽略异常
    }
}
```

## 代码示例

### 带冲突检测的下载

```csharp
var apiService = new ModrinthAPIService();
var analysisService = new ModAnalysisService();
var downloadService = new ModDownloadService(apiService, analysisService);

// 构建冲突回调
ModConflictCallback conflictCallback = async (info) =>
{
    Console.WriteLine($"检测到冲突: {info.ModName}");
    Console.WriteLine($"本地版本: {info.ExistingVersion}");
    Console.WriteLine($"新版本: {info.NewVersion}");
    
    // 在实际应用中弹出对话框让用户选择
    return ModConflictResolution.Replace;
};

// 带冲突检测的下载
var results = await downloadService.DownloadWithDependenciesAsync(
    "fabric-api", "1.20.1", "fabric", @"C:\mods",
    VersionType.Release, progress, cancellationToken,
    conflictCallback);
```

### 无冲突检测的下载（兼容旧用法）

```csharp
// 不传 conflictCallback 参数即跳过冲突检测
var results = await downloadService.DownloadWithDependenciesAsync(
    "fabric-api", "1.20.1", "fabric", @"C:\mods",
    VersionType.Release, progress, cancellationToken);
```

### 使用 LocalModInventory 独立扫描

```csharp
var analysisService = new ModAnalysisService();
var inventory = new LocalModInventory(@"C:\mods", analysisService);
await inventory.ScanAsync();

// 查看已安装的模组
foreach (var mod in inventory.InstalledMods)
{
    Console.WriteLine($"{mod.Name} v{mod.Version} ({mod.LoaderType})");
}

// 查询特定模组
var fabricApi = inventory.FindByModId("fabric-api");
if (fabricApi != null)
    Console.WriteLine($"已安装: {fabricApi.Name} v{fabricApi.Version}");
